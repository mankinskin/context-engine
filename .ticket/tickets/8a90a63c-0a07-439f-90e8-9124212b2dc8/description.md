Umbrella program for new store domains and operational quality loops extending the cross-store architecture.

Vectors:
- interview store for editable interview sessions and multi-user survey orchestration
- feedback store as high-volume structured inbox with search and reconciliation workflows
- continuous health auditing for stale/conflicting/unhelpful spec/rule/ticket entries
- rule-generated instruction/skill scaffolding for bootstrapping new domain stores from a single prompt

Acceptance criteria:
- each vector has a tracker ticket with implementation and validation tickets
- planning specs define contracts, metrics, and migration constraints
- dependencies and health are clean for implementation handoff
