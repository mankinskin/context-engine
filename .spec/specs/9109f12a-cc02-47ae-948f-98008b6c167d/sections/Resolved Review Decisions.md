- **Q2.1 / Q2.3 Index-file provenance prefixes are distinct from content-file prefixes.** Index/catalog files always use an `-index`/`-catalog` suffixed prefix so they are never confused with content files that already emit `rule-api:*` / `spec` provenance. Concretely: rule catalog uses `rule-catalog:file` / `rule-catalog:entry`; spec index uses `spec-index:file` / `spec-index:entry`; ticket/audit/workspace use `ticket-index:*`, `audit-index:*`, `workspace-index:*`.
- **Q2.2 Canonical snippet block structure (fixed here, not per-generator).** Each entry renders as: a `### <title>` heading, then the one-line normalized summary, then an optional bullet list for keywords/tags/relations. Generators may add domain-specific bullets but must not change this skeleton. This guarantees a consistent Tier-0/1/2 LOD structure across domains.
- **Q4.1 Entry marker carries a digest prefix.** The per-entry provenance comment is extended to `<!-- <prefix>:entry id=<uuid> slug=<slug> digest=<first-12-hex> -->`. This makes the peek Tier-0 line free (no extra body line) and greppable by `id` or `digest`. R2 is amended to include the `digest=` field.
